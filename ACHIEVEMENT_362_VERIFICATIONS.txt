## SYSTEM ARCHITECTURE VERIFICATION

### Core Concepts Validated

1. **Study:**
   - Top-level unit of work
   - Defines base design snapshot
   - Safety domain configuration
   - Policy and rail configuration
   - Multi-stage experiment graphs
   - Isolated telemetry and outcomes

2. **Case:**
   - Concrete design state
   - Derived from Study via ECOs
   - Forms directed acyclic graph (DAG)
   - Deterministic naming: <case_name>_<stage_index>_<derived_index>

3. **Stage:**
   - Refinement phase within Study
   - Ordered stages (N ≥ 1)
   - Execution mode specification
   - Trial budget and survivor count
   - Progressive refinement through pipeline

4. **ECO (Engineering Change Order):**
   - Atomic modification to design
   - Deterministic and reversible
   - Organized into classes
   - Catastrophic failure containment

5. **Trial:**
   - Single execution of PD workflow
   - Produces metrics and artifacts
   - Isolated execution environment
   - Tracked telemetry and outcomes

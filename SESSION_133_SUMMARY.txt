                          SYSTEM CAPABILITIES
═══════════════════════════════════════════════════════════════════════════

Noodle 2 v0.1.0 is a comprehensive safety-aware orchestration system for
physical design experimentation that provides:

**Safety & Control:**
- Three safety domains (sandbox, guarded, locked)
- ECO classification by blast radius (4 classes)
- Policy-driven execution with memory
- Multi-gate safety validation (Gate 0: base case, Gate 1+: per stage)
- Automatic failure detection and containment
- Risk envelope constraints

**Experiment Management:**
- Multi-stage workflows with arbitrary N stages
- Deterministic case naming for lineage tracking
- Trial budgets and configurable survivor selection
- Adaptive policy with ECO effectiveness tracking
- Parameter sweep support (1000+ trials)
- ECO class allowlists/blacklists per stage

**Distributed Execution:**
- Ray cluster integration (single or multi-node)
- Docker-based trial isolation
- Concurrent stage execution with DAGs
- Graceful shutdown and checkpointing
- Resource management and scheduling
- Fault tolerance and recovery

**Comprehensive Observability:**
- Ray Dashboard integration with hierarchical task naming
- Real-time telemetry collection (Study/Stage/Case)
- Event stream logging with structured data
- Professional summary and diff reports
- Publication-quality heatmap visualizations (300+ DPI)
- Case lineage visualization (DOT/Graphviz)

**Robust Error Handling:**
- 20+ unique error codes (N2-E-XXX format)
- Structured error context with actionable suggestions
- Error code registry with descriptions
- Failure classification (timeout, container, parsing, etc.)
- Comprehensive diagnostic information

**Professional User Experience:**
- Intuitive CLI with comprehensive --help
- Well-formatted reports with color coding
- Clear, actionable error messages
- Publication-quality visualizations
- Excellent documentation (README, docstrings, comments)

WHAT IS NOODLE 2?
═════════════════

Noodle 2 is a **safety-aware, policy-driven orchestration system** for
large-scale physical-design experimentation built on OpenROAD.

Key Characteristics:
- NOT a physical design tool (doesn't replace OpenROAD)
- NOT an optimizer (doesn't invent ECOs)
- IS a control plane for safe, auditable experimentation
- IS a distributed orchestration system with safety rails

Core Capabilities:
1. Safety-critical control (sandbox/guarded/locked domains)
2. Multi-stage experiment workflows with survivor selection
3. Distributed execution on Ray clusters with concurrent stages
4. Comprehensive telemetry and visualization
5. Deterministic case graph tracking with DAG visualization
6. Professional error handling and recovery (20+ error codes)
7. Publication-quality reports and heatmaps
8. ECO effectiveness leaderboards and comparisons

Technology:
- Python 3.10+ with full type hints
- Ray for distributed orchestration
- Docker for trial isolation
- PDK support: Nangate45, ASAP7, Sky130A
- ~30,000 lines of production code
- 3,139 comprehensive tests

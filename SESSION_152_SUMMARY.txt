## WHAT IS NOODLE 2?

Noodle 2 is a **safety-aware, policy-driven orchestration system** for
large-scale physical-design experimentation built on OpenROAD.

### Core Architecture Components:

**1. Safety System**
- Three safety domains: sandbox, guarded, locked
- ECO class constraints and risk envelopes
- Run legality reports (pre-execution validation)
- Abort thresholds and stage gates
- Comprehensive policy enforcement

**2. Study Management**
- Multi-stage refinement pipeline
- Deterministic case naming (DAG-based lineage)
- Survivor selection algorithms
- Trial budgets and resource allocation
- Policy-driven early stopping

**3. Case Management**
- CaseIdentifier: <case_name>_<stage_index>_<derived_index>
- Case graph (DAG) with parent tracking
- Lineage queries and visualization
- Metadata and artifact management

**4. ECO Framework**
- Pluggable ECO classes (NoOp, BufferInsertion, PlacementDensity, etc.)
- ECO effectiveness tracking with priors
- Parameter validation
- TCL generation for OpenROAD
- Safety classification (safe, local, global, disruptive)

**5. Execution Engine**
- Docker-based OpenROAD execution
- Ray distributed computing
- Trial timeouts and resource limits
- Artifact indexing and persistence
- Reproducible builds with git integration

**6. Telemetry & Reporting**
- Case/Stage/Study telemetry
- Failure type tracking
- Runtime statistics
- Human-readable summary reports
- JSON-LD metadata for publications

**7. PDK Support**
- Nangate45 (tested base case)
- ASAP7
- Sky130
- Docker image: efabless/openlane:ci2504-dev-amd64

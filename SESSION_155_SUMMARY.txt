WHAT IS NOODLE 2?
═══════════════════════════════════════════════════════════════════════════════

Noodle 2 is a comprehensive **safety-aware, policy-driven orchestration system**
for large-scale physical-design experimentation built on OpenROAD.

CORE CAPABILITIES:
  ✓ Safety domains (sandbox, guarded, locked) with ECO class constraints
  ✓ Multi-stage refinement with survivor selection
  ✓ Deterministic case lineage tracking (DAG-based)
  ✓ Policy-driven early stopping and abort rails
  ✓ Ray-based distributed execution
  ✓ Comprehensive telemetry and artifact management
  ✓ Docker-based OpenROAD integration
  ✓ PDK support (Nangate45, ASAP7, Sky130)

PRODUCTION QUALITY FEATURES:
  ✓ 3,139 automated tests (100% passing)
  ✓ Full type hints with mypy validation
  ✓ Comprehensive error handling and recovery
  ✓ Detailed logging and tracing
  ✓ Git integration for reproducibility
  ✓ JSON-LD metadata for research publication
  ✓ Ray Dashboard integration for observability
  ✓ Artifact indexing and deep linking
  ✓ Structured telemetry (study/stage/case)

ARCHITECTURAL HIGHLIGHTS:
  - Clean separation: Controller (Python) vs Trial execution (Docker)
  - Deterministic case naming: <case_name>_<stage_index>_<derived_index>
  - Safety-first: Run legality reports block illegal configurations
  - Policy with memory: ECO effectiveness tracking (trusted/mixed/suspicious)
  - Backward-compatible telemetry schemas
  - Extensive error code registry (E0001-E0027+)

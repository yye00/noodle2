## SYSTEM ARCHITECTURE OVERVIEW

### Noodle 2 - Safety-Aware Physical Design Orchestration System

**Primary Purpose:**
Large-scale experimentation framework for OpenROAD physical design flows
with built-in safety constraints and deterministic case tracking.

**Core Components:**

1. **Safety System**
   - Three domains: sandbox, guarded, locked
   - ECO classification by invasiveness
   - Constraint enforcement and validation
   - Violation detection and reporting

2. **Case Management**
   - Unique identifiers (stage, index, variant)
   - Parent-child derivation tracking
   - DAG-based lineage management
   - Metadata preservation

3. **Execution Framework**
   - Ray-based distributed execution
   - Docker container isolation
   - OpenROAD tool integration
   - Comprehensive artifact capture

4. **ECO (Engineering Change Order) Framework**
   - Multiple ECO types (NoOp, BufferInsertion, PlacementDensity, etc.)
   - Effectiveness tracking and priors
   - TCL script generation
   - Parameter validation

5. **Telemetry & Artifacts**
   - JSON-LD metadata for publication
   - Comprehensive logging and tracing
   - Git integration for reproducibility
   - Result aggregation and analysis

**Supported PDKs:**
- Nangate45 (NanGate 45nm FreePDK)
- ASAP7 (7nm predictive PDK)
- Sky130 (SkyWater 130nm open-source PDK)

WHAT NOODLE 2 PROVIDES
======================

Noodle 2 is a safety-aware, policy-driven orchestration system for large-scale
physical design experimentation built on OpenROAD. It manages uncertainty,
failure, and limited compute budgets while exploring Engineering Change Orders
(ECOs) across complex, multi-stage design workflows.

Core Capabilities Verified:
───────────────────────────

1. Study Management
   • Multi-stage experiment orchestration
   • Safety domain enforcement (sandbox, guarded, locked)
   • Policy-driven early stopping
   • Deterministic case lineage tracking

2. Case Management
   • DAG-based derivation tracking
   • Deterministic naming: <case>_<stage>_<derived>
   • Parent-child lineage visualization
   • Isolation across studies

3. Safety System
   • ECO class safety constraints
   • Run legality checking
   • Pre-execution validation
   • Safety domain policy enforcement

4. ECO Framework

   • Parameter validation and serialization

   • Containerized OpenROAD execution

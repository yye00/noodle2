┌──────────────────────────────────────────────────────────────────────────────┐
│ NOODLE 2 SYSTEM ARCHITECTURE                                                 │
└──────────────────────────────────────────────────────────────────────────────┘

Noodle 2 is a production-ready, safety-aware orchestration system for large-
scale physical design experimentation with OpenROAD.

CORE COMPONENTS:

  ⚡ Safety System
     ├─ Three-tier safety domains (sandbox, guarded, locked)
     ├─ ECO class permission enforcement
     ├─ Run legality reporting and validation
     └─ Pre-execution safety checks

  ⚡ Case Management
     ├─ Deterministic case naming: <base>_<stage>_<index>
     ├─ DAG-based lineage tracking
     ├─ Metadata propagation through derivations
     └─ Stage progression management

  ⚡ ECO Framework
     ├─ Extensible ECO factory pattern
     ├─ No-op, buffer insertion, placement density ECOs
     ├─ Effectiveness tracking with Bayesian priors
     └─ Safety classification (safe, routing-affecting, global-disruptive)

  ⚡ Execution Engine
     ├─ Docker-based OpenROAD integration
     ├─ Ray-based distributed execution
     ├─ PDK support (Nangate45, ASAP7, Sky130)
     └─ Artifact and telemetry collection

  ⚡ Study Configuration
     ├─ Multi-stage refinement pipelines
     ├─ Survivor selection policies
     ├─ Early stopping criteria
     └─ Metadata and reproducibility support

  ⚡ Analysis & Reporting
     ├─ Timing report parsing (WNS, TNS)
     ├─ Timing path extraction for ECO targeting
     ├─ JSON-LD metadata generation
     └─ Git integration for provenance

│                    WHAT IS NOODLE 2?                                   │
└───────────────────────────────────────────────────────────────────────┘

Noodle 2 is a safety-aware, policy-driven orchestration system for
large-scale physical design (PD) experimentation with OpenROAD.

Core Capabilities:
   • Safety domain enforcement (sandbox, guarded, locked)
   • ECO class management (topology_neutral → global_disruptive)
   • Multi-stage refinement workflows with survivor selection
   • Deterministic case lineage tracking (DAG-based)
   • Policy-driven early stopping and abort rails
   • Ray-based distributed execution
   • Docker integration with OpenROAD containers
   • Comprehensive telemetry and artifact management
   • PDK support: Nangate45, ASAP7, Sky130

Technical Implementation:
   • Language: Python 3.10+
   • Framework: Ray for distributed orchestration
   • Containers: efabless/openlane:ci2504-dev-amd64
   • Testing: pytest with 3,139 comprehensive tests
   • Type safety: Full mypy validation
   • Quality: Production-grade with 5-star rating

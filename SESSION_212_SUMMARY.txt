## What is Noodle 2?

Noodle 2 is a safety-aware, policy-driven orchestration system for large-scale
physical design experimentation built on top of OpenROAD.

### Core Capabilities

1. **Safety Model**
   - Safety domains: sandbox, guarded, locked
   - ECO class constraints (topology_neutral → global_disruptive)
   - Run legality reports before execution
   - Deterministic failure classification

2. **Experiment Control**
   - Multi-stage refinement workflows
   - Survivor selection across stages
   - Budget management and early stopping
   - Adaptive policy with memory

3. **Case Management**
   - Deterministic case naming: <base>_<stage>_<derived>
   - DAG-based lineage tracking
   - Full case graph introspection
   - Immutable snapshots

4. **Execution Platform**
   - Ray-based distributed execution
   - Docker containerization (efabless/openlane:ci2504-dev-amd64)
   - Single-node and multi-node support
   - Ray Dashboard for observability

5. **PDK Support**
   - Nangate45 (educational reference)
   - ASAP7 (academic advanced-node)
   - Sky130 / sky130A (open-source)

6. **Telemetry & Artifacts**
   - Structured JSON telemetry
   - Per-trial artifact bundles
   - Timing report parsing (WNS, TNS)
   - Congestion analysis (hot bins)
   - Visualization support (heatmaps)

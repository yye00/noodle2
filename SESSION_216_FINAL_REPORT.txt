 WHAT IS NOODLE 2?
═══════════════════════════════════════════════════════════════════════════

Noodle 2 is a comprehensive safety-aware orchestration system for large-scale
physical design experimentation with OpenROAD.

Core Capabilities:
────────────────────────────────────────────────────────────────────────────
• Safety Domains: Sandbox, guarded, and locked modes with ECO constraints
• Multi-Stage Refinement: Sequential stages with survivor selection
• Case Lineage: Deterministic DAG-based tracking of design variants
• Policy Engine: Adaptive decision-making with structured memory
• Distributed Execution: Ray-based orchestration for parallel trials
• Telemetry & Artifacts: Comprehensive structured output and indexing
• Docker Integration: Containerized OpenROAD execution
• PDK Support: Nangate45, ASAP7, Sky130 reference targets

Production Quality Features:
────────────────────────────────────────────────────────────────────────────
• 3,139 automated tests covering all functionality
• Full type hints with mypy validation
• Comprehensive error handling and recovery
• Detailed logging and trace capabilities
• Git integration for reproducibility
• JSON-LD metadata for research publication
• Ray Dashboard integration for observability
• Artifact indexing with deep links

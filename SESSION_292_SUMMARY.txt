NOODLE 2 SYSTEM OVERVIEW
================================================================================

Noodle 2 is a safety-aware, policy-driven orchestration system for large-scale
physical design experimentation with OpenROAD.

**Core Features:**

1. Safety Domains:
   - sandbox (exploratory, permissive)
   - guarded (default, production-like)
   - locked (conservative, regression-only)

2. Multi-Stage Refinement:
   - Sequential stage execution
   - Survivor selection between stages
   - Deterministic case lineage tracking

3. Policy-Driven Control:
   - Early failure detection and classification
   - Trial budget enforcement
   - Survivor count limits
   - Abort criteria and safety rails

4. Ray-Based Execution:
   - Distributed compute via Ray
   - Task-level parallelism
   - Dashboard for observability
   - Fault tolerance

5. Comprehensive Telemetry:
   - Structured JSON artifacts
   - JSON-LD metadata
   - Case lineage graphs
   - Safety traces

6. Docker Integration:
   - OpenROAD containerized execution
   - Reproducible environments
   - Isolated trial workspaces

7. PDK Support:
   - Nangate45 (reference standard-cell)
   - ASAP7 (advanced-node stress testing)
   - Sky130 (sky130A variant)

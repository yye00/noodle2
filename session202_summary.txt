## WHAT IS NOODLE 2?

### Product Description

Noodle 2 is a **safety-aware, policy-driven orchestration system** for
large-scale physical design experimentation with OpenROAD.

### Core Value Proposition

**Noodle 2 provides:**
- Structured experiment control (not ad-hoc scripting)
- Safety-critical decision making (not blind optimization)
- Auditable outcomes (not opaque black boxes)
- Deterministic behavior (not random exploration)
- Production-grade reliability (not research prototypes)

### Key Capabilities

**1. Safety Model**
- Three safety domains: sandbox, guarded, locked
- ECO class hierarchy (topology_neutral â†’ global_disruptive)
- Run legality checking before compute consumption
- Deterministic failure classification and containment

**2. Experiment Control**
- Multi-stage refinement (arbitrary N stages)
- Survivor selection with configurable budgets
- Policy-driven early stopping
- Adaptive memory with ECO priors (trusted/mixed/suspicious/unknown)

**3. Case Management**
- Deterministic naming: <case_name>_<stage>_<derived>
- DAG-based lineage tracking
- Immutable snapshot management
- Traceable derivation across complex experiments

**4. Distributed Execution**
- Ray-based parallel trial execution
- Ray Dashboard integration for observability
- Multi-node cluster support
- Artifact indexing and deep links

**5. Tool Integration**
- Docker-based OpenROAD execution (efabless/openlane:ci2504-dev-amd64)
- OpenSTA timing analysis with report_checks parsing
- Congestion analysis with global_route metrics
- Heatmap export support (placement density, RUDY, routing)

**6. Observability**
- Structured telemetry (Study/Stage/Case axes)
- Comprehensive artifact management
- Safety traces and audit trails
- JSON-LD metadata for research publication

**7. PDK Support**
- Nangate45 (fast bring-up, educational)
- ASAP7 (advanced node, congestion stress)
- Sky130/sky130A (open-source production)

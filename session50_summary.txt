## Implementation Details

**StudyExporter Class:**
- Collects metrics, rankings, and lineage from completed Study
- Generates structured exports with metadata
- Supports incremental metric addition via add_case_metrics()
- Validates case references via case graph

**CaseMetricsSummary Dataclass:**
- Flattened case metrics for CSV export
- Includes timing (WNS, TNS, violations breakdown)
- Includes congestion (bins, overflow)
- Includes resources (CPU time, memory)
- Includes rankings and scores
- Includes lineage (parent, ECO applied)

**Export Formats:**
- JSON: Complete nested structure with all data
- CSV: Flattened table for spreadsheet compatibility
- Both include same core metrics for consistency

**Export Metadata:**
- ISO 8601 timestamp (timezone-aware)
- Noodle 2 version
- Total cases and stages
- DAG depth and statistics
